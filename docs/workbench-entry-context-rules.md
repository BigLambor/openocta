# 运维工作台入口上下文规则说明

## 1. 核心定义与设计原则

运维工作台的“技术域上下文”是决定工作台展现哪些运维对象、场景目录和 AI 数字助手的核心依据。为了避免用户认知混乱，必须对各入口上下文进行严格的规范：

1. **工作台主导航直接进入**：默认进入“全部技术域” (all)。
2. **场景/告警/巡检触发的下钻跳转**：必须严格携带源头的业务域（如 `hadoop`、`gbase`、`fi` 等），并不允许静默退回到“全部技术域”。
3. **数据传递一致性**：URL 参数中的 `?domain=xxx`、技术域选择器选中项以及持久化存储（LocalStorage/SessionState）需实现完全一致。

## 3. 技术实现细节与辅助函数

### 3.1 统一深链辅助逻辑 `openWorkbench`
工作台不推荐手动拼接路由。当前实现中，`app-render.ts` 内部通过 `openWorkbench(opts)` 统一处理入口跳转：

* `domain`：有明确来源域时写入全局技术域上下文；顶部导航无来源域时保留为 `all`。
* `view`：显式传入事件、巡检、诊断、治理、容量或变更中心；未传时默认进入事件中心。
* `scenarioId`：从域详情场景卡进入时写入 `workbenchScenario`，并重置对象范围为 `all`。
* `alertGroupId`：从告警入口进入时写入对应域的 `opsSelectedAlertGroupIds[domain]`。
* 进入工作台或切换技术域时，会清理场景搜索、成熟度过滤和旧对象范围，避免跨域残留。

全域场景下，助手面板展示“全域值班数字员工”；只有选中具体告警或进入具体技术域场景时，才显示对应技术域数字员工。

### 3.2 历史退回与状态持久化规则
* 当用户从工作台返回其他 Tab 时，`selectedOpsDomain` 不会被清空，以便再次点击工作台时能恢复上一次的临时上下文状态。
* 单个技术域的告警选中 ID 分别存储在 `state.opsSelectedAlertGroupIds[domain]` 中，确保在切换技术域时，不会由于跨域 ID 冲突导致详情面板加载报错。
* Flink/Spark 作业对象范围使用作业 `id` 作为 scope 值，页面 label 显示作业名称，避免跨集群同名作业误选。
* HDFS `/tmp`、`/user`、`/app` 目录入口必须带集群和 namespace 上下文，是静态治理热点目录，不代表实时目录树枚举。
