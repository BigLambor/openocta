# 导航、场景与侧边栏改造任务清单

> 日期：2026-06-05  
> 来源：`docs/navigation_and_scenario_analysis.md`、`docs/sidebar_design_analysis.md` 以及当前架构评审结论  
> 目标：修复驾驶舱下钻语义、明确业务场景落位，并为运维工作台/服务与资产补齐专业侧边栏。

## 0. 总体原则

| 原则 | 说明 |
|------|------|
| 驾驶舱负责“看” | 看全局态势、域健康、场景风险，不承载复杂操作 |
| 域详情负责“看清问题” | 作为驾驶舱下钻页，展示域内场景健康和快速入口 |
| 工作台负责“做” | 事件、巡检、诊断、治理、容量、变更的日常闭环 |
| 服务与资产负责“管对象” | 集群、服务、组件、作业、拓扑、责任关系和标签 |
| AI 贯穿工作流 | AI 不独立成中心，作为驾驶舱/域详情/工作台的上下文分析能力 |
| 技术域是上下文过滤器 | 不再恢复“每个技术域复制一套能力菜单”的旧结构 |

状态约定：

```text
⬜ 待开始
🟡 进行中
✅ 已完成
⚠️ 阻塞
❌ 不通过
```

## 推荐实施顺序

1. **先做 P0**：修复驾驶舱“打开运维域”的跳转语义。
2. **再做 P2/P3**：补工作台和资产页侧边栏，立刻解决左侧空白和导航效率问题。
3. **再做 P1**：新增域详情页，承接驾驶舱下钻和场景展示。
4. **再做 P4**：逐步迁移 Flink/Spark/HDFS 业务场景。
5. **最后做 P5/P6**：AI 上下文标准化和收藏/最近访问/折叠等增强。

## 最小可验收版本

如果要快速落一个可评审版本，建议只做以下 8 项：

| ID | 任务 |
|----|------|
| N0-1 | 调整驾驶舱域卡片主按钮语义 |
| N0-2 | 增加“进入域详情 / 查看资产”双入口 |
| S2-1 | 新增通用侧边栏组件 |
| S2-2 | 工作台布局改造 |
| S2-3 | 工作台能力模块导航 |
| S2-5 | 工作台技术域过滤迁入侧边栏 |
| S3-1 | 资产页布局改造 |
| S3-3 | 资产类型导航迁入侧边栏 |

验收通过后，再推进域详情页和 BCH 场景迁移。
