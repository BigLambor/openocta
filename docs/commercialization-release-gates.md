# 商用发布验收门禁

> 状态：Phase 0 基线 v1  
> 适用范围：试点版、私有化交付版、正式商用版发布前检查

## 必过门禁

| 分类 | 最低门槛 |
|------|----------|
| 数据 | `openocta.db` migration 幂等；核心对象无新增 JSON 主写；旧 JSON 导入有备份和幂等测试。 |
| Demo 数据 | 默认启动不写示例集群、示例告警；演示环境必须显式设置 `OPENOCTA_SEED_DEMO_DATA=1`。 |
| 安全 | 无默认可登录账号；密码强哈希；生产 token 不依赖 localStorage 明文长期凭证。 |
| 权限 | HTTP 与 WebSocket 主业务方法有权限声明；Agent 工具执行不能绕过高风险审批。 |
| 审计 | 资产、告警、任务、审批、配置、高风险工具执行写审计日志。 |
| 任务闭环 | 手动、Cron、告警触发的执行记录可追溯 run、steps、tool/model 调用和结果。 |
| 备份恢复 | SQLite 备份和恢复命令可在空环境恢复并通过 readyz。 |
| 可观测 | `/healthz`、`/readyz`、结构化日志可用；关键错误含 request/run/session 关联。 |
| 测试 | migration、权限越权、审批、E2E 商用闭环、备份恢复测试通过。 |

## 发布阻断项

- 默认写入 demo seed 数据。
- 资产、告警、任务、审批、权限、健康结果新增 JSON 主写路径。
- migration 失败后仍以部分可写状态启动。
- 高风险工具或配置变更无审批、无审计。
- 备份产物无法恢复到空环境。

## 发布记录要求

每次商用候选版本必须记录：

| 字段 | 内容 |
|------|------|
| 版本 | 应用版本、schema version、前端构建版本。 |
| 数据状态 | migration 起止版本、旧 JSON 导入数量、备份路径。 |
| 安全状态 | 管理员初始化方式、认证方式、权限策略版本。 |
| 测试状态 | 必过测试命令、E2E 结果、未覆盖风险。 |
| 运维状态 | 备份验证时间、readyz 结果、部署形态。 |
