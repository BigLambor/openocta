# 运维工作台 & 服务与资产 左侧空白区域分析

## 现状诊断

### 问题本质

从代码和截图分析，左侧空白不是"缺了一个组件"，而是**页面布局策略**的问题：

| 页面 | 有侧边栏？ | 导航方式 | 空间利用 |
|------|:---:|---------|---------|
| AI 运维助手 | ✅ | 左侧会话列表 | 充分 |
| 运维驾驶舱 | ❌ | 无需导航（单页仪表盘） | 充分（卡片撑满） |
| **运维工作台** | ❌ | **顶部 Tab**（6个模块） | ⚠️ **左侧空白** |
| **服务与资产** | ❌ | **顶部 Tab**（5个资产类型） | ⚠️ **左侧空白** |
| 系统管理 | ✅ | 左侧菜单树 | 充分 |
| 自动化配置 | ✅ | 左侧菜单树 | 充分 |

> [!IMPORTANT]
> **核心问题**：运维工作台和服务与资产都是**功能密集型**页面（多层级、多维度），仅靠顶部 Tab 无法承载完整的导航层级，导致：
> 1. 左侧大量留白，空间浪费
> 2. 导航层级扁平化，子功能入口不明显
> 3. 与有侧边栏的页面（AI助手、系统管理）体验不一致

---

## 行业对标

| 产品 | 类似页面 | 左侧侧边栏用法 |
|------|---------|--------------|
| **Datadog** | Infrastructure / APM | 左侧：服务树 + 过滤器（环境、团队、标签）|
| **Grafana** | Alerting / Dashboards | 左侧：文件夹树 + 状态过滤 |
| **ServiceNow** | ITSM Workspace | 左侧：模块菜单 + 收藏 + 最近访问 |
| **PagerDuty** | Service Directory | 左侧：服务层级 + 团队过滤 |
| **Dynatrace** | Smartscape | 左侧：实体树（主机、进程、服务） |
| **阿里云 ARMS** | 应用监控 | 左侧：应用列表 + 环境筛选 |

> [!TIP]
> **行业共识**：运维类平台的工作页面几乎都采用"**左侧导航 + 右侧内容**"的布局，因为运维场景天然需要在**多个资源/对象之间快速切换**。

---

## 方案建议

### 方案 A（推荐）：为工作台和资产页增加上下文侧边栏

每个页面的侧边栏内容不同，体现各自的导航需求：

### 方案 B：可折叠侧边栏 + 域资产树

类似 VS Code 或 IDE 的可折叠面板风格：

```
┌─ 折叠态 ─┐  ┌─ 展开态 ─────────────────┐
│  📂      │  │  📂 技术域                │
│  🔍      │  │  ├─ BCH 生态              │
│  ⭐      │  │  │  ├─ hadoop-prod-01     │
│  🕐      │  │  │  │  ├─ HDFS           │
│           │  │  │  │  ├─ YARN           │
│           │  │  │  │  ├─ Flink          │
│           │  │  │  │  └─ Spark          │
│           │  │  │  └─ hadoop-prod-02     │
│           │  │  ├─ FI 商业生态           │
│           │  │  └─ ...                   │
```

**特点**：
- 折叠后只显示图标（44px），不占内容空间
- 展开后显示完整树形导航
- 用户可自由控制

## 推荐方案：方案 A

> [!IMPORTANT]
> **推荐方案 A 的核心理由**：
> 1. **体验一致性**——AI 助手和系统管理已经有侧边栏，工作台和资产页也加上后，整个应用的布局语言统一
> 2. **导航效率**——运维工程师需要在多个模块/域/资产之间频繁切换，侧边栏比顶部 Tab 效率高得多
> 3. **信息增量**——侧边栏可以展示 Tab 无法承载的信息：角标计数、树形层级、收藏、最近访问
> 4. **行业标准**——Datadog、Grafana、ServiceNow 等主流运维平台全部采用侧边栏布局

### 实现优先级

| 优先级 | 任务 | 说明 |
|:---:|------|------|
| P0 | 运维工作台侧边栏 | 6 大模块导航 + 技术域过滤器 + 角标计数 |
| P0 | 服务与资产侧边栏 | 域-资产树形导航 + 资产计数 + 搜索 |
| P1 | 收藏视图 & 最近访问 | 工作台侧边栏的个性化快捷区域 |
| P1 | 标签过滤 | 服务与资产侧边栏的多维度筛选 |
| P2 | 侧边栏折叠/展开 | 支持用户自定义宽度或折叠成图标栏 |

### 设计规范

```
┌──────────────────────────────────────────────────────────┐
│  Header (56px)                                            │
├──────────┬───────────────────────────────────────────────┤
│          │                                               │
│ Sidebar  │  Main Content                                 │
│ (240px)  │  (flex: 1)                                    │
│          │                                               │
│ - 可选折叠│  - padding: 24px                              │
│ - 分隔线  │  - max-width: none (满宽)                     │
│ - 滚动独立│  - overflow-y: auto                           │
│          │                                               │
└──────────┴───────────────────────────────────────────────┘

Sidebar 规范:
- 宽度: 240px (折叠态: 44px)
- 背景: var(--bg-sidebar) 或 slightly darker than content
- 边框: right 1px solid var(--border-color)
- 内边距: 12px
- 字体: 13px
- 分组标题: 11px, uppercase, color: var(--text-tertiary)
- 选中项: background highlight + left border accent
```
